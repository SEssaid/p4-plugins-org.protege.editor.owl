OWL Editor

This is one of the core Protégé plugins and is responsible for creating the core owl editing environment.

Version info

v4.1.0 Update Build 220 - 24 Mar, 2011

  • Fixed a refresh problem in the property hierarchy panels where deleting a property did not remove it from the property hierarchy.
  • Fixed a bug where dragging a class in the class hierarchy to its own superclass would remove the class from the tree.
  • In these cases, ontologies on the web would be slow to load because Protege did make good use of content negotiation. This has been fixed.
  • Imports are not automatically redirected to the users disk if the choice of file to choose for the import is ambiguous.
  • Fixed some incompatibilities involving how different copies of identical axioms are displayed. An axiom can appear more than once in an ontology if it is imported from different locations or if the two copies of the same axiom have different annotations.
